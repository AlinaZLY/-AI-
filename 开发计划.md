# 基于 AI 的校园招聘服务平台 — 开发计划

## 一、项目概述

- **项目名称**：基于 AI 的校园招聘服务平台
- **团队规模**：4 人
- **版本管理**：Git 分支协作（feature → develop → fix/integration → main）

---

## 二、团队分工与功能详情（按分支划分）

### `feature/alina-ai-interview` — 张力尹（@AlinaZLY）

**课题**：基于AI的校园招聘服务平台的设计与实现——智能面试与评估模块

| 子模块 | 功能点 |
|---|---|
| **1.1 AI 模拟面试** | 岗位 JD 解析：自动提取岗位要求的核心技能、经验、学历等 |
| | 简历解析：提取简历中的项目经历、技能点 |
| | 生成面试题：根据 JD + 简历智能生成针对性问题 |
| | 答题模式：支持语音答题、文字答题 |
| | AI 评分：从内容完整性、逻辑性、专业性等维度评分 |
| | 改进建议：给出答案优化建议和参考回答 |
| **1.2 模拟面试记录** | 历史记录：保存每次模拟面试记录 |
| | 面试复盘：查看历史答题和评分 |
| | 能力雷达图：可视化展示各维度能力 |
| **1.3 面试题库** | 系统题库：内置常见面试题（宝洁八大问、技术题等） |
| | 自定义题库：用户自主添加遇到的面试题 |
| | 分类管理：按岗位、公司、题型分类 |
| | 题目标签：难度、频率、类型标签 |

---

### `feature/siyuan-resume-ai` — 王思远（@siyuanWang348）

**课题**：基于AI的校园招聘服务平台的设计与实现——简历管理与AI优化模块

## 四、开发阶段规划

### 第一阶段：基础架构搭建（王俊翔主导，全员配合）
- [ ] **王俊翔**在 `feature/junxiang-community-user` 分支搭建 NestJS 后端框架
  - 项目脚手架初始化
  - MySQL + Redis 连接配置
  - 公共模块（数据库模块、认证模块、统一响应格式、异常处理）
  - 用户注册 / 登录 / JWT 鉴权接口
- [ ] 数据库设计（ER 图、表结构）
- [ ] 统一代码规范与提交规范
- [ ] **完成后合入 `develop`，通知全员拉取**

### 第二阶段：各模块并行开发
登录系统合入 `develop` 后，各人从 `develop` 拉取最新代码，然后并行开发：

- [ ] `feature/junxiang-community-user`：用户中心 → 社区论坛（帖子/评论/点赞/收藏/搜索/推荐） → 消息通知
- [ ] `feature/siyuan-resume-ai`：简历上传/多版本管理 → 模板/编辑/预览 → AI 分析 → AI 优化
- [ ] `feature/junyi-delivery-tracking`：投递记录 CRUD → 状态流转 → 时间线/日历 → 筛选搜索 → 备注
- [ ] `feature/alina-ai-interview`：面试题库管理 → JD + 简历解析 → 面试题生成 → 答题模式 → AI 评分 → 记录/雷达图

### 第三阶段：合入 develop 联调
按依赖顺序合入：
1. **王俊翔**先合（用户中心 + 社区）
2. **王思远**合（简历模块，面试模块依赖简历数据）
3. **张峻屹**合（投递追踪，相对独立）
4. **张力尹**最后合（AI 面试，依赖最多）

联调问题在 `fix/integration` 分支修复。

### 第四阶段：完善与发布
- [ ] 各模块细节打磨
- [ ] Bug 修复与性能优化
- [ ] 合入 `main` 正式发布
- [ ] 毕设论文与答辩准备

### 关键协作节点

| 时间点 | 事件 | 涉及人员 |

## 五、模块间依赖关系

```
用户登录系统 (模块 3.3)
    ├── 模块 1: 智能面试（需要用户身份）
    │       └── 依赖模块 2 的简历解析数据
    ├── 模块 2: 简历管理（需要用户身份）
    ├── 模块 3: 社区论坛（需要用户身份）
    └── 模块 4: 投递追踪（需要用户身份）
```

**关键依赖链**：
1. 模块 3 的用户登录系统 → 所有模块（最先开发）
2. 模块 2 的简历解析 → 模块 1 的面试题生成（需协调接口）
3. 模块 1 和模块 4 的面试记录功能有重叠，需明确边界：
   - 模块 1：AI 模拟面试的记录与评分
   - 模块 4：真实面试流程的投递状态追踪与备注
