# 本地开发配置教程

## 1. 环境

- Git
- Node.js 20+
- MySQL
- Redis

## 2. Git 配置

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

## 3. 配 SSH

```bash
ssh-keygen -t ed25519 -C "你的邮箱"
cat ~/.ssh/id_ed25519.pub
ssh -T git@github.com
```

如果你用了 SSH 别名，测试时也用对应 Host，例如：

```bash
ssh -T github-alinazly
```

公钥添加到：

`https://github.com/settings/keys`

## 4. 第一次拉代码

```bash
git clone git@github.com:AlinaZLY/-AI-.git
cd -AI-
git fetch --all --prune
git branch -a
```

如果你本机用了 SSH 别名，按 `~/.ssh/config` 里的 Host 写地址，例如：

```bash
git clone git@github-alinazly:AlinaZLY/-AI-.git
```

## 5. 拉自己的分支

先执行：

```bash
git fetch --all --prune
```

Alina：

```bash
git switch -c feature/alina-ai-interview --track origin/feature/alina-ai-interview
```

思源：

```bash
git switch -c feature/siyuan-resume-ai --track origin/feature/siyuan-resume-ai
```

峻屹：

```bash
git switch -c feature/junyi-delivery-tracking --track origin/feature/junyi-delivery-tracking
```

俊翔：

```bash
git switch -c feature/junxiang-community-user --track origin/feature/junxiang-community-user
```

更新当前分支：

```bash
git pull
```

## 6. 拉完整代码

本项目约定：

- `main`：合并好的完整代码
- `feature/alina-ai-interview`：Alina 的分支
- `feature/siyuan-resume-ai`：思源的分支
- `feature/junyi-delivery-tracking`：峻屹的分支
- `feature/junxiang-community-user`：俊翔的分支

如果要拿完整项目，直接切 `main`：

```bash
git fetch --all --prune
git switch main
git pull origin main
```

## 7. 装依赖

```bash
cd server
npm install
cd ../admin
npm install
cd ../client
npm install
```

## 8. 配后端

```bash
cd ../server
Copy-Item .env.example .env
```

`.env` 至少配置：

```env
PORT=3000
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=你的数据库密码
DB_DATABASE=campus_recruitment
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
```

## 9. 导数据库

```sql
CREATE DATABASE campus_recruitment DEFAULT CHARSET utf8mb4;
```

导入：

`server/sql/campus_recruitment_dump.sql`

## 10. 启动

```bash
cd server
npm run start:dev
```

```bash
cd admin
npm run dev
```

```bash
cd client
npm run dev
```

## 11. 地址

- 后端：`http://127.0.0.1:3000`
- 管理端：`http://localhost:3100`
- 用户端：`http://localhost:5173`

## 12. 常见问题

- `Permission denied (publickey)`：SSH 没配好
- 多 GitHub 账号时不要写死 `git@github.com:`，要用自己 `~/.ssh/config` 里的 Host 别名
- 分支切不过去：先执行 `git fetch --all --prune`
- `3306` 连不上：MySQL 没启动
- `6379` 连不上：Redis 没启动
